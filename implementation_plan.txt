# Quiet Hours Generator Automation (Genset Controller) - Implementation Plan

Implement a reliable, warranty-safe automation system for a Kirloskar diesel generator using a Sedemac KG645 controller. The primary goal is to prevent the generator from automatically starting during nighttime power outages (Quiet Hours) while preserving all native KG645 AMF (Automatic Mains Failure) functionality during the day.

## Aligned Design Decisions

* **Relay Default-OFF Fail-Safe**: On ESP8266 reboot, power loss, or startup, the relay **MUST** default to **OFF** (Normally Open contacts disconnected). This ensures that any ESP8266 failure or power loss automatically restores the generator's factory AMF behavior, preventing the generator from being permanently disabled.
* **Non-Blocking WiFi/Blynk Connection**: Standard Blynk examples use `Blynk.begin()`, which blocks execution if WiFi or Blynk servers are unreachable. We implemented a custom, non-blocking connection manager to ensure the ESP8266 remains responsive and fails safe even if the local WiFi is down.
* **Direct Generator Status Monitoring**: The HW-622 board's built-in PC817 optocoupler allows direct monitoring of the generator's 12V status output without requiring external level-shifting hardware.
* **Resistor Hack**: Solved the relay activation failure by soldering three 220Ω resistors in series (660Ω) in parallel across the onboard transistor base resistor R6 (4.7kΩ).
* **Auto / Manual Interlocking Switch (V3)**: Added a dedicated Auto Mode toggle switch (`V3`) on Blynk. Turning it ON puts the controller in schedule-driven mode. Toggling the relay manually (`V4`) automatically disables Auto Mode (`V3` turns OFF) to preserve the manual override.

## Final Hardware Configuration & Wiring Plan

### 1. ESP8266 WiFi Relay Board Layout
The ESP8266-12E WiFi Relay Module (HW-622) uses the following pin configurations:
* **Relay Output**: Controlled by **GPIO4** (Active HIGH).
* **Onboard LED**: **GPIO2** (Active LOW).
* **Onboard Sensor Input**: **GPIO5** (Connected to the built-in opto-isolator output to monitor generator running state).

### 2. Transistor Base Resistor Mod (Resistor Hack)
* **Problem**: The onboard relay driver transistor (Q1) base resistor (R6) was 4.7kΩ. Driven by the ESP8266's 3.3V GPIO4, it provided only 0.55mA of base current—insufficient to fully saturate the transistor. The collector voltage was stuck at 1.9V, leaving only 3.1V across the 5V relay coil, causing it to fail to click.
* **Solution**: Soldered **three 220Ω through-hole resistors in series** (yielding **660Ω**) in parallel across **R6** (tacked between the transistor base and GPIO4 output trace).
* **Result**:
  * Combined base resistance ($R_{eq}$): **~578Ω** (Calculated as $\frac{4700 \times 660}{4700 + 660}$)
  * Current drawn from ESP8266 GPIO4: **~4.5mA** (safe, recommended limit is 12mA)
  * Transistor collector voltage drops to **~0.1V**, and the relay clicks and latches perfectly.

### 3. Direct Generator Status Monitoring (Opto-Isolated GPIO5)
* **Design**: The HW-622 board contains a built-in PC817 optocoupler (U3) with a 4.7kΩ current-limiting resistor (R8) on the input side (`INPUT_1`). We bypassed the need for any external optocoupler module.
* **Wiring**:
  * **KG645 Terminal 3 (OUT A)** ───► **`IN` (INPUT_1)** (Board 3-pin input block)
  * **KG645 Terminal 1 (BATT-)** ───► **`GND` (GND_EXIT)** (Board 3-pin input block)
* **Operation**:
  * KG645 `OUT A` is configured as **Generator Available (Gen Available)**, active-HIGH (12V on run, 0V on stop).
  * 12V applied to the board input draws **2.3mA** through the built-in 4.7kΩ resistor, safely triggering the built-in optocoupler and pulling **GPIO5 LOW** in code when the generator is running. No external level shifters or external resistors are required.

### 5. KG645 Controller Programmed Settings
Configure the following parameters in the KG645 controller using the front keypad or the KG645 Configuration Utility:
*   **Digital Input B (Simulate Mains)**:
    *   **Source (SOURCE)**: Set to `14: Simulate Mains` (confirms to the controller that mains voltage is healthy).
    *   **Polarity (POLARITY)**: Set to `Close To Activate` (active when connected to BATT-).
    *   **Action (ACTION)**: Set to `None`.
*   **Digital Output A (Generator Available Status)**:
    *   **Source (SOURCE)**: Set to `28: Generator available (Gen Available)`.
    *   **On Activation (ON ACTIVATION)**: Set to `Energise` (outputs BATT+ / 12V to drive the opto-isolator input when running).

## Final Software Architecture

The software is implemented in the Arduino framework inside `genset_controller.ino`.

### A. Core Software Features
1. **Auto / Manual Interlocking Controls**:
   * **V3 (Auto Mode)** enables time-based scheduler enforcement.
   * **V4 (Simulate Mains)** controls the physical relay (`RELAY_PIN = 4`). Toggling this manually automatically turns off Auto Mode.
   * **V2 (LED Switch)** controls the physical blue LED (`LED_PIN = 2`). The logic is active-low (`LOW` = LED ON, `HIGH` = LED OFF).
2. **NTP Fallback Scheduler**:
   * Synchronizes UTC+5:30 local time via SNTP.
   * If NTP has successfully synchronized at least once, the scheduler executes locally on the chip's internal clock (`millis()`) even if the connection is lost.
   * Quiet Hours are set between **11:00 PM and 6:00 AM**. If `autoMode` is enabled and the clock falls within this window, the controller activates the relay.
3. **Non-Blocking Connection Management**:
   * WiFi connections are managed dynamically using the **WiFiManager** library with a captive portal portal timeout of 180 seconds.
   * Reconnection attempts to WiFi and Blynk are decoupled and run on timers to ensure the `loop()` is completely non-blocking, protecting local scheduling.

### B. Verified Bug Fixes
* **NTP Client Rate-Limiting**: The `timeClient.update()` returns `false` when it does not make a network request due to its internal 60-second rate-limiting. The scheduler was updated to run based on the synchronized epoch time rather than block on `update()`'s boolean return value, preventing error spam and allowing full offline schedule tracking.
